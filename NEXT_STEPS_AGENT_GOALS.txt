# FERRITE: NEURAL INFERENCE FRAMEWORK
## Goal: Transform Ferrite into a multi-crate Rust framework for sandboxed GPU inference.

### CURRENT SYSTEM ARCHITECTURE
- **Logic Layer (WASM Guest)**: Orchestrates the prompt flow, provides the user interface (CLI chat), and handles high-level decision making. 
- **Inference Layer (Native Host)**: A high-performance Rust runner using `candle` to execute GGUF models on NVIDIA GPUs via CUDA. 
- **The Bridge (WIT)**: A WebAssembly Interface Type contract (`ferrite.wit`) that allows the WASM guest to call the Host's "Neural Muscle" without knowing anything about CUDA.

### KEY COMPONENTS
1. `ferrite-runtime`: The current host implementation using `wasmtime` and `candle`.
2. `wasm-examples/mistral-inference`: A working 4-bit quantized Mistral bot running inside WASM.
3. `ferrite.wit`: The interface definition defining how models are loaded and used.

### THE NEXT MISSION (FOR THE NEXT AGENT)
Your goal is to refactor this repository from a "working prototype" into a "publishing-ready framework."

#### 1. CRATE REFACTORING
Currently, the runtime is mixed with the app logic. We need:
- `ferrite-core`: Pure model inference logic (GGUF, CUDA adapters).
- `ferrite-wasm-host`: A library crate that other developers can import to add "Neural WASM" support to their own Rust apps. It should contain the `Wasmtime` orchestration.
- `ferrite-sdk`: The crate that WASM guest authors use (currently `ferrite-guest`).

#### 2. IMPROVE AUTHENTICATION & DX
- We recently added support for Hugging Face tokens to access gated models (like Mistral v0.2).
- Goal: Make token handling more "automatic." Look at checking `~/.cache/huggingface/token` or environment variables by default so the user doesn't have to hardcode tokens in their `lib.rs`.

#### 3. STREAMING & ASYNC
- The current WIT interface is synchronous and returns full strings.
- Goal: Implement real-time token streaming across the WASM boundary using WASI streams or an iterator-based WIT pattern.

#### 4. MODEL MANAGEMENT
- The host handles downloads via `hf-hub`.
- Goal: Add a `ModelManager` that can track local inventory, verify checksums, and allow the WASM guest to query "Which models do I have available?"

### STATUS CHECK
- Mistral 7B Q4 is verified working at ~60 tokens/sec.
- WASM-to-Host bridge is stable.
- Hugging Face cache wiping and authenticated downloads are working.

**Direction: "Make AI logic portable by wrapping the dirty work of GPU math in a clean, sandboxed WASM interface."**
